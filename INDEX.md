## Quick Start Guide

### For New Clients
1. Navigate to `Clients/` folder
2. Copy the `_Template` folder
3. Rename to client name
4. Fill out `CLAUDE.md` with client details
5. Begin organizing project materials

### For New Projects
1. Create project folder under `Clients/[ClientName]/Projects/`
2. Use standard structure: Documentation, Deliverables, Data, Scripts
3. Add project details to client's master plan.md

### For Knowledge Management
1. Add reusable learnings to `Knowledge Base/`
2. Document new tools and processes
3. Keep industry insights updated
4. Link related documents together

## File Organization Principles

### Naming Conventions
- Use descriptive, clear names
- Date format: `YYYY-MM-DD`
- Separate words with hyphens or underscores
- Keep file names concise but meaningful

### Version Control
- Use git for code and documentation
- Keep archives of major versions
- Document changes in project logs

### Document Management
- Add "Last Updated" dates to knowledge documents
- Link related files together
- Maintain README/INDEX files in each major folder
- Regular cleanup of outdated materials
